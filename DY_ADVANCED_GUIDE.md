## 功能详解

### 1. 策略回测 📊

**功能说明**：
- 在历史数据上测试策略表现
- 模拟真实交易过程
- 计算收益率、胜率、盈亏比等指标

**使用步骤**：
1. 选择策略（激进/稳健/最强）
2. 设置回测天数（30-365天）
3. 点击"开始回测"
4. 查看回测结果

**回测指标**：
- 总收益率：策略的总体收益百分比
- 交易次数：触发的交易数量
- 胜率：盈利交易占比
- 平均盈利/亏损：单笔交易的平均盈亏
- 盈亏比：平均盈利与平均亏损的比值
- 最终资金：初始 $100,000 的最终金额

**示例结果**：
```
策略：稳健策略
总收益率：15.8%
交易次数：25
胜率：68%
平均盈利：$1,250
平均亏损：$-450
盈亏比：2.78
最终资金：$115,800
```

### 2. 策略对比 📈

**功能说明**：
- 同时对比三种策略
- 可视化图表展示
- 全面的性能对比

**使用步骤**：
1. 点击"对比三种策略"
2. 等待回测完成
3. 查看对比图表和表格

**对比维度**：
- 总收益率
- 交易次数
- 胜率
- 盈亏比

**图表类型**：
- 柱状图：直观对比收益率
- 表格：详细数据对比

**使用场景**：
- 选择最适合自己的策略
- 了解不同策略的特点
- 根据市场环境调整策略

---

### 3. 自定义策略 🎯

**功能说明**：
- 创建个性化筛选策略
- 保存和管理多个策略
- 灵活组合筛选条件

**创建步骤**：
1. 输入策略名称
2. 填写策略描述
3. 选择筛选条件：
   - 买入/卖出信号
   - UP1/UP2/UP3 趋势
   - DOWN1/DOWN2/DOWN3 趋势
4. 点击"创建策略"

**筛选条件说明**：
- **买入信号**：底背离 + MACD 金叉
- **卖出信号**：顶背离 + MACD 死叉
- **UP1**：突破蓝带上轨
- **UP2**：突破黄带上轨
- **UP3**：蓝带突破黄带（强势）
- **DOWN1-3**：对应的下跌信号

**示例策略**：

```
策略名称：超强势策略
描述：买入 + UP2 + UP3，双重确认
条件：✓ 买入信号  ✓ UP2  ✓ UP3
```

```
策略名称：风险规避
描述：卖出 + 任意下跌，及时止损
条件：✓ 卖出信号  ✓ DOWN1  ✓ DOWN2  ✓ DOWN3
```

**管理策略**：
- 查看已保存的策略列表
- 应用策略到当前结果
- 删除不需要的策略

---

### 4. 历史追踪 📝

**功能说明**：
- 记录每日筛选信号
- 追踪信号后续表现
- 建立历史数据库

**使用步骤**：

**保存信号**：
1. 在主页完成筛选
2. 切换到"历史追踪"标签
3. 点击"保存当前信号"
4. 系统记录所有信号到数据库

**查看历史**：
1. 点击"查看历史"
2. 浏览历史信号记录
3. 按日期、股票筛选

**数据库结构**：
```sql
signals 表：
- date: 日期
- symbol: 股票代码
- price: 价格
- buy/sell: 买卖信号
- up1/up2/up3: 上涨趋势
- down1/down2/down3: 下跌趋势

signal_performance 表：
- signal_id: 关联信号
- days_after: 信号后第几天
- price: 当时价格
- return_pct: 收益率
```

**应用场景**：
- 验证信号有效性
- 优化策略参数
- 建立交易日志

---

### 5. 性能分析 📉

**功能说明**：
- 分析策略历史表现
- 统计关键指标
- 可视化展示

**使用步骤**：
1. 选择要分析的策略
2. 点击"分析表现"
3. 查看统计结果和图表

**分析指标**：
- 总信号数：历史触发次数
- 平均收益：所有信号的平均收益率
- 胜率：盈利信号占比
- 最大收益：单次最高收益
- 最大亏损：单次最大亏损
- 中位数收益：收益的中位数
- 标准差：收益的波动性

**图表展示**：
- 收益分布图
- 胜率趋势图
- 累计收益曲线

**使用建议**：
- 定期分析策略表现
- 对比不同时期的数据
- 根据分析结果调整策略

## API 接口文档

### 回测相关

#### POST /api/backtest
运行策略回测

**请求**：
```json
{
  "symbols": ["AAPL", "MSFT"],
  "strategy": "stable",
  "days": 180
}
```

**响应**：
```json
{
  "success": true,
  "result": {
    "strategy": "stable",
    "total_return_pct": 15.8,
    "total_trades": 25,
    "win_rate_pct": 68.0,
    "avg_win": 1250.0,
    "avg_loss": -450.0,
    "profit_factor": 2.78,
    "final_capital": 115800.0
  }
}
```

#### POST /api/compare_strategies
对比多个策略

**请求**：
```json
{
  "symbols": ["AAPL", "MSFT"],
  "strategies": ["aggressive", "stable", "strongest"],
  "days": 180
}
```

### 自定义策略相关

#### GET /api/custom_strategies
获取策略列表

**响应**：
```json
{
  "success": true,
  "strategies": [
    {
      "name": "我的策略",
      "description": "买入 + UP1"
    }
  ]
}
```

#### POST /api/custom_strategies
创建策略

**请求**：
```json
{
  "name": "我的策略",
  "description": "买入 + UP1",
  "conditions": {
    "buy": true,
    "up1": true
  }
}
```

#### DELETE /api/custom_strategies/{name}
删除策略

#### GET /api/apply_custom_strategy/{name}
应用策略

### 历史追踪相关

#### POST /api/save_signals
保存当前信号

**请求**：
```json
{
  "date": "2026-03-04"
}
```

#### GET /api/signal_history
获取历史信号

**参数**：
- start_date: 开始日期
- end_date: 结束日期
- symbol: 股票代码

#### GET /api/strategy_performance/{strategy}
获取策略表现

**参数**：
- days: 持有天数
- start_date: 开始日期
- end_date: 结束日期

## 注意事项

### 回测注意事项
1. **历史表现不代表未来**：回测结果仅供参考
2. **数据质量**：免费数据可能有延迟或错误
3. **过拟合风险**：避免过度优化参数
4. **交易成本**：回测未考虑手续费和滑点

### 自定义策略注意事项
1. **条件不宜过多**：太多条件会减少机会
2. **逻辑要合理**：确保条件组合有意义
3. **定期review**：市场变化需要调整策略
4. **记录原因**：记录创建策略的理由

### 历史追踪注意事项
1. **每日保存**：养成每日保存信号的习惯
2. **数据备份**：定期备份数据库文件
3. **清理旧数据**：定期清理过期数据
4. **隐私保护**：不要分享包含交易记录的数据库

## 常见问题

**Q: 回测需要多长时间？**
A: 取决于股票数量和回测天数，通常 5-15 分钟。

**Q: 可以回测自定义策略吗？**
A: 当前版本暂不支持，计划在下个版本添加。

**Q: 历史数据保存在哪里？**
A: 保存在 `signal_history.db` SQLite 数据库文件中。

**Q: 如何导出历史数据？**
A: 可以使用 SQLite 工具导出，或通过 API 获取后导出为 CSV。

**Q: 策略对比的股票列表是什么？**
A: 默认使用前 20 只 S&P 500 成分股，可以自定义。
