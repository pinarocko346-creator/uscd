# 🎉 问题已解决！

## 问题原因

macOS系统的 **AirPlay Receiver** 服务默认占用了5000端口，导致Flask无法使用该端口。

## 解决方案

已将Web服务器端口从 **5000** 改为 **8080**。

## ✅ 现在可以访问了！

### 启动服务器

```bash
cd ~/three-layer-quant-system
python web_server.py
```

### 访问地址

在浏览器中打开以下任一地址：

- **http://localhost:8080**
- **http://127.0.0.1:8080**

### 快速启动

```bash
./start_web.sh
```

## 📱 其他设备访问

如果想从手机或其他设备访问：

1. 查看你的Mac IP地址：
   ```bash
   ifconfig | grep "inet " | grep -v 127.0.0.1
   ```

2. 在其他设备浏览器访问：
   ```
   http://你的IP地址:8080
   ```

## 🔧 如果还是打不开

### 检查1：确认服务器正在运行

```bash
# 查看是否有Python进程在运行
ps aux | grep web_server.py
```

### 检查2：测试端口是否可访问

```bash
curl http://localhost:8080
```

如果返回HTML内容，说明服务器正常。

### 检查3：查看防火墙设置

确保防火墙没有阻止8080端口。

### 检查4：查看服务器日志

运行服务器时会显示详细日志，查看是否有错误信息。

## 💡 端口说明

- **原端口**：5000（被AirPlay占用）
- **新端口**：8080（常用的Web开发端口）
- **可选端口**：如果8080也被占用，可以改为其他端口（如8000、3000等）

## 🎯 修改端口方法

如果需要使用其他端口，编辑 `web_server.py` 最后一行：

```python
app.run(debug=True, host='0.0.0.0', port=8080)  # 改为你想要的端口
```
