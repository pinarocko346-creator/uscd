# DY 选股器 Web 界面使用指南

## 快速启动

### 方式一：使用启动脚本（推荐）

```bash
cd /Users/apple/three-layer-quant-system
./start_dy_web.sh
```

### 方式二：直接运行

```bash
cd /Users/apple/three-layer-quant-system
python3 dy_web_server.py
```

启动后，在浏览器中访问：**http://localhost:5001**

## 界面功能

### 1. 筛选参数设置

- **最低股价**：过滤低于此价格的股票（默认 $0.1）
- **最低交易额**：过滤日交易额低于此金额的股票（默认 $10,000,000）
- **并发线程数**：同时处理的股票数量，越大越快但占用资源越多（默认 10）
- **股票来源**：
  - 自动获取：从 S&P 500 和 NASDAQ 100 获取（约 600 只）
  - 自定义：手动输入股票代码，用逗号分隔

### 2. 开始筛选

点击"🚀 开始筛选"按钮后：
1. 系统会显示实时进度条
2. 显示当前处理阶段（过滤 → 分析）
3. 显示已处理的股票数量

### 3. 查看结果

筛选完成后会显示：

#### 统计卡片
- 总股票数
- 买入信号数量
- 卖出信号数量
- 上涨趋势数量
- 下跌趋势数量

#### 筛选标签
- **全部**：显示所有结果
- **🟢 买入信号**：只显示有买入信号的股票
- **🔴 卖出信号**：只显示有卖出信号的股票
- **📈 上涨趋势**：只显示有上涨趋势的股票
- **📉 下跌趋势**：只显示有下跌趋势的股票

#### 结果表格
显示每只股票的：
- 股票代码
- 当前价格
- 买卖信号（BUY/SELL）
- 趋势信号（UP1/UP2/UP3/DN1/DN2/DN3）

### 4. 导出数据

点击"📥 导出 CSV"按钮可以将结果导出为 CSV 文件，方便进一步分析。

然后在结果中筛选"买入信号"，找到有底背离反转机会的低价股。

## 信号说明

### 买入信号 🟢
- 出现底背离（价格新低但 MACD 未新低）
- MACD DIFF 上穿 DEA（金叉）
- **建议**：关注这些股票的买入机会

### 卖出信号 🔴
- 出现顶背离（价格新高但 MACD 未新高）
- MACD DIFF 下穿 DEA（死叉）
- **建议**：如果持有这些股票，考虑减仓

### 趋势信号

**上涨趋势**：
- UP1：突破蓝带上轨（短期突破）
- UP2：突破黄带上轨（长期突破）
- UP3：蓝带突破黄带（强势信号）

**下跌趋势**：
- DN1：跌破蓝带下轨（短期破位）
- DN2：跌破黄带下轨（长期破位）
- DN3：蓝带跌破黄带（弱势信号）

## 最佳实践

### 1. 分阶段筛选

```
第一步：宽松条件快速筛选
  → 最低股价: 0.1
  → 最低交易额: 5000000
  → 获得候选股票池

第二步：关注买入信号
  → 点击"买入信号"标签
  → 查看有底背离的股票

第三步：结合趋势确认
  → 优先选择同时有 UP1/UP2/UP3 的股票
  → 这些是最强势的买入机会
```

### 2. 定期监控

```
每日收盘后运行一次筛选
  → 发现新的买入机会
  → 监控持仓股票是否出现卖出信号
  → 导出 CSV 进行历史对比
```

### 3. 风险控制

```
不要只依赖单一信号
  → 买入信号 + 上涨趋势 = 更可靠
  → 结合基本面分析
  → 设置止损止盈
```

## 常见问题

### Q: 筛选速度慢怎么办？
A: 增加并发线程数（建议 15-20），但注意不要超过 50，否则可能被数据源限流。

### Q: 某些股票无法获取数据？
A: 这是正常的，可能是股票代码错误、退市或数据源暂时不可用。系统会自动跳过这些股票。

### Q: 可以筛选其他市场的股票吗？
A: 当前版本只支持美股。如需支持其他市场，需要修改数据源。

### Q: 结果可以保存吗？
A: 可以点击"导出 CSV"按钮保存结果，也可以在浏览器中截图保存。

### Q: 可以同时运行多个筛选任务吗？
A: 不可以，系统一次只能运行一个筛选任务。等待当前任务完成后再启动新任务。

## 性能优化建议

1. **首次运行**：建议先用少量股票测试（自定义 10-20 只）
2. **网络环境**：确保网络稳定，VPN 可能影响数据获取速度
3. **系统资源**：并发线程数不要超过 CPU 核心数的 2 倍
4. **数据缓存**：yfinance 会自动缓存数据，重复筛选会更快

## 停止服务

在终端中按 `Ctrl+C` 即可停止 Web 服务。
